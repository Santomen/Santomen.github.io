UP-VID

Es un software diseñado para la Universidad Panamericana cuando se preveía un posible regreso a clases presenciales, este software ayuda a organizar la entrada de los alumnos a la universidad, teniendo en cuenta la cantidad permitida de alumnos en cada salón, permitiendo que los alumnos sean capaces de reservar clases una vez por semana por materia igualmente dándole la oportunidad de cancelar las reservaciones, además si algún alumno se reporta enfermo de inmediato se le revocaría el permiso para asistir a clases presenciales hasta que su salud se recupere, así mismo da aviso a los alumnos con los que compartió clase de que asistieron a clase con un alumno que reportó tener la enfermedad COVID-19, este software no tiene una base de datos para registrar la información, tiene en cambio archivos csv donde se almacena el conjuto de datos.

Proceso

Para realizar este proyecto se utilizó c++, hubo ciertos problemas al intentar guardar la información en los archivos y recuperarla ya que no se utilizó una base de datos, además de que se usaron 2 archivos csv y estos de alguna manera debían de estar conectados, por lo que se utilizó un algoritmo diseñado por el equipo para recuperar, guardar y hacer las modificaciones correspondientes a la información conforme se van modificando los datos.

Lógica del código

Estructura “credencial” para guardar la información del alumno (nombre, ID, edad, campus, carrera).
Clase “estudiante” para guardar información del alumno dentro de la estructura y extra (contraseña, semestre, día de asistencia, info de COVID).
Clase “salon” para guardar la información del salón (materia, capacidad, ubicación, presencial o no, disponibilidad por COVID, número de clase, ID del salón, lugares disponibles, lugares registrados).

Un poco sobre las funciones del código

Dentro del programa hay 2 tipos de usuarios, los alumnos y el administrador, el primero solo tiene permisos para hacer reservaciones siempre que haya espacio disponible en el salón y en el día escogido , puede realizar cancelaciones de clase siempre y cuando tenga al menos una clase reservada, puede actualizar constantemente su estado de salud, puede imprimir sus datos, mientras que el segundo usuario puede agregar salones al software así como modificar las propiedades de este como: si está disponible para ser ocupado en clases presenciales, modificar la capacidad permitida de alumnos del salón en cuestión.
